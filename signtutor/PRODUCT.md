# SignTutor — Product Context

## Users
- Deaf and hard-of-hearing individuals learning ASL
- Hearing people learning to communicate with Deaf community
- Students, educators, and interpreters in training
- Privacy-conscious users who refuse cloud-based sign recognition

## Brand
- **Name:** SignTutor
- **Tagline:** Learn ASL with real-time, on-device feedback
- **Tone:** Professional, warm, intelligent, accessible
- **Anti-references:** Cold clinical dashboards, SaaS-cream gradients, generic ed-tech templates

## Product Purpose
A browser-based ASL learning app that uses MediaPipe Hands and ONNX ML to give real-time fingerspelling and sign feedback entirely in the browser. No video uploads, no accounts, no cloud processing. Privacy-first by design.

## Strategic Principles
1. **On-device by default** — all ML runs locally, no data leaves the device
2. **Accessibility first** — WCAG 2.2 AA, keyboard navigation, screen reader support
3. **Progressive learning** — FSRS spaced repetition, adaptive difficulty
4. **Hands are the hero** — everything celebrates human hand communication

## Tech Stack

- Tailwind CSS

- ONNX Runtime Web (ML inference)
