Product Design Requirement · v2 (Refined)

Vision-Based Sign Language Learning Platform

Webcam-only · On-device inference · Co-designed with Deaf advisors
Draft for review

What it is

A browser-based tutor that teaches sign language with a watch → sign → get feedback loop. Inference runs on-device, so webcam frames never leave the user's machine by default. The MVP ships one language end-to-end with a multi-language selector scaffolded for future variants.

  • Static fingerspelling (alphabet & digits)
  • Dynamic isolated signs (25 in MVP → 200 in v1 → 1,000+ in v2)
  • Structured feedback: handshape · location · movement · palm orientation
  • Non-manual markers (facial expression) — v2, behind feature flag

Architecture at a glance

  • Capture: getUserMedia → canvas
  • Landmarks: MediaPipe Holistic (hands · pose · face)
  • Static signs: MLP on hand landmarks + temporal smoothing
  • Dynamic signs: sliding-window Transformer / ST-GCN
  • Runtime: ONNX Runtime Web / TF.js, WebGPU when available
  • Server: static CDN + optional account/sync, no video upload

Privacy & accessibility (non-negotiables)

  • Webcam frames & landmarks never leave the device by default; persistent "🔒 On-device" indicator.
  • WCAG 2.2 AA; keyboard-only nav; captions/transcripts on every video.
  • Mirroring on by default; left-handed mode flips demo and recognition.
  • COPPA/FERPA-ready flows for minors and schools.
  • One-click account & data deletion.

Fairness & community

  • Deaf advisory board with veto over curriculum and terminology.
  • Paid, credited native signers for all demo videos.
  • Evaluation slices: skin tone (Monk scale), handedness, lighting, camera angle.
  • Fairness gate: no slice > 5 pp below overall accuracy → release blocker.

Top changes from PDR v1

  • Replaced HMM as headline model with landmark-based Transformer / ST-GCN; HMM kept as evaluation baseline.
  • Added personas, user stories, success metrics, MVP/v1/v2 scope columns, and a 6-milestone roadmap.
  • Added first-class privacy (on-device by default), accessibility (WCAG 2.2 AA), and fairness (subgroup gates) sections.
  • Added structured feedback taxonomy (handshape · location · movement · orientation · NMM) instead of binary right/wrong.
  • Added data strategy: bootstrap datasets, in-house native-signer recording, opt-in user contributions.
  • Added Deaf advisory board as a release gate.
