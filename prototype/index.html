Architecture (PDR §7.1, S-05): webcam → MediaPipe Hands → 21×3 landmarks → normalize() (a JS port of ml/features.py, byte-equivalent contract) → ONNX MLP (sklearn-trained, exported to ONNX, loaded with onnxruntime-web) → softmax → top-3 + confidence-gated verdict. The rule-based classifier is still here as a toggle (so you can compare) and its per-finger output drives the actionable tips ("extend your thumb"), because the MLP itself is opaque on which finger is wrong. A startup self-test runs the model on a canonical input and logs PyTorch ↔ ONNX parity (line in the log below). All inference is on-device.

Note: this preview iframe has no network, so MediaPipe + onnxruntime-web can't download. Open the file via localhost to use it for real; if the model fails to load, the page falls back to the rule-based classifier so the UI still works.
