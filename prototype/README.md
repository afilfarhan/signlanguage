# SignTutor — Prototype Bundle

Four working pages, two ML pipelines (static MLP + dynamic Transformer), and a
CI workflow that enforces the PDR's release gates on every push.

```
prototype/
├── home.html           # entry point — links the four flows
├── wizard.html         # camera-setup wizard (PDR S-01)
├── index.html          # STATIC fingerspelling — ONNX MLP, 24 letters, per-finger panel ⭐
├── dynamic.html        # DYNAMIC isolated signs — ONNX Transformer (8 signs)
├── educator.html       # v2 educator dashboard wireframe (EPIC-15)
└── models/
    ├── fingerspell_mlp.onnx              # 72 KB, 24 classes
    ├── labels.json                       # labels + synth-collision groups
    ├── parity_samples.json
    ├── dynamic_signs_transformer.onnx    # 360 KB
    └── dynamic_labels.json

ml/
├── synth_landmarks.py / synth_sequences.py
├── features.py / seq_features.py
├── train_export.py             # 24-class MLP, collision-aware eval
├── train_transformer.py        # Transformer (--rotation-aug closes NFR-10 gate)
├── eval_robustness.py
├── ROBUSTNESS_GATE_DEMO.md
└── models/                     # ONNX + reports + before/after snapshots

ci/
├── check_nfr.py                # parses reports, enforces NFR-9 / NFR-10 / parity gates
├── run_parity.mjs              # JS↔Python feature-parity check (Node, no deps)
└── parity.log                  # last parity run output (also uploaded as CI artifact)

.github/workflows/
└── ml-gate.yml                 # runs everything above on push / PR
```

## How to run

### Serve the prototype
```
cd prototype && python3 -m http.server 8000
# open http://localhost:8000/home.html
```

### Reproduce the CI gate locally
```
pip install numpy scikit-learn skl2onnx onnx onnxruntime onnxscript \
            torch --index-url https://download.pytorch.org/whl/cpu

python3 ml/train_export.py
python3 ml/train_transformer.py --rotation-aug
python3 ml/eval_robustness.py

cp ml/models/{fingerspell_mlp.onnx,labels.json,dynamic_signs_transformer.onnx,dynamic_labels.json} prototype/models/

# Regenerate parity samples
python3 - <<'PY'
import json, sys, numpy as np; sys.path.insert(0, 'ml')
from synth_landmarks import _synthesize_one
from features import normalize
rng = np.random.default_rng(42); samples = {}
for letter in ['B','F','I','L','Y']:
    lm = _synthesize_one(letter, rng); feats = normalize(lm)
    samples[letter] = {
        'landmarks': [[round(float(v), 6) for v in pt] for pt in lm],
        'expected_normalized_first6': [round(float(v), 6) for v in feats[:6]],
        'expected_normalized_l2': round(float(np.linalg.norm(feats)), 6),
    }
json.dump(samples, open('prototype/models/parity_samples.json','w'), indent=2)
PY

node ci/run_parity.mjs | tee ci/parity.log
python3 ci/check_nfr.py    # exits non-zero on any gate violation
```

## What each page does

### `wizard.html` — camera-setup wizard (S-01)
4-step onboarding: language → permission → lighting check (live in-browser
luminance + uniformity meters; no frames uploaded) → framing & handedness.
Persists prefs to `localStorage.signtutor.prefs`.

### `index.html` — static-sign lesson player (S-05) ⭐
**End-to-end ML pipeline, now covering all 24 static ASL letters** (J & Z are
motion → dynamic page).

```
getUserMedia
  ↓
MediaPipe Hands (WASM)         →  21×3 landmarks
  ↓
JS normalize()                 ←  byte-equivalent port of ml/features.py
  ↓
ONNX MLP via onnxruntime-web   (sklearn, 72 KB, 24 classes)
  ↓
top-5 probabilities + confidence-gated verdict
  ↓
structured feedback panel
+ per-finger panel (rule-derived ✓/✗ vs target)
+ single most-actionable tip
```

**New since the last increment:**
- **24-letter curriculum.** All static letters from `synth_landmarks.py`'s
  `LETTERS` list. Includes per-letter formation hints in the demo card.
- **Per-finger panel.** Five compact tiles (Thumb · Index · Middle · Ring ·
  Pinky) showing whether each finger is currently extended (✓) or curled (✗),
  *colored by whether it matches the target letter's pattern*. Greens up when
  the user gets it right, reds when not. Same data the rule-based tip
  generator already used, surfaced visually.
- **Collision-aware verdict.** When the model's top prediction is in the same
  *synth-collision group* as the target (e.g. it predicts `E` and you wanted
  `A` — both are closed-fist patterns the synth genuinely cannot tell apart),
  the verdict shows `close` with a friendly explanation rather than `miss`.
- **Top-5 (was top-3).** With 24 classes, top-5 is more useful.

**Hybrid classifier preserved.** Toggle ML ↔ Rule; ML auto-falls-back to Rule
when the model can't load (e.g. the sandboxed iframe preview). The rule layer
is also the source of the per-finger panel and the per-finger tips.

**Startup self-test.** Loads `parity_samples.json` (5 canonical landmark
frames generated by `_synthesize_one(seed=42)`) and checks: (1) JS
`normalize()` outputs match Python within 1e-3 (first-6 features and L2 norm);
(2) the ONNX model's top-1 matches ground truth on each. Result is logged.

### `dynamic.html` — dynamic-sign lesson player (S-06)
MediaPipe → 45-frame landmark buffer → JS sequence normalization (port of
`ml/seq_features.py`) → 360 KB Transformer (ONNX). Two operating modes
(explicit record vs. continuous).

### `educator.html` — v2 educator dashboard wireframe (EPIC-15)

## CI release gates ⭐

`.github/workflows/ml-gate.yml` runs on every push to `ml/`, `prototype/`,
or `ci/`:

1. Train the static MLP, write `eval_report.md`.
2. Train the dynamic Transformer with `--rotation-aug`, write
   `dynamic_eval_report.md`.
3. Run the robustness sweep, write `dynamic_robustness_report.md`.
4. Sync the trained models into `prototype/models/`.
5. Regenerate `parity_samples.json` from the trained pipeline.
6. Run `ci/run_parity.mjs` (Node) — verifies JS `normalize()` matches Python.
7. Run `ci/check_nfr.py` — parses the reports and **exits non-zero** if:
   - **NFR-9**: static accuracy < 0.85 (collision-aware) OR dynamic < 0.80
   - **NFR-10**: any robustness slice deviates more than 5pp from `clean`
   - **parity**: JS↔Python feature drift exceeds tolerance
8. Uploads all reports as CI artifacts so reviewers can read them on the PR.

**Verified locally that the gate works both ways:**
- ✅ PASSES on the augmented dynamic model (rotation slice 0.972, ▼1.9pp).
- ❌ FAILS on the pre-augmentation snapshot (rotation slice 0.906, ▼9.4pp),
  with a precise error message naming the offending slice and delta.

## Path to v1

1. Replace synthetic data with real datasets. Strict accuracy will jump as
   the model gains finger-curvature/orientation cues that resolve the synth
   collision groups; collision-aware accuracy will stay above NFR-9.
2. Expand dynamic to 25 → 200 signs (Transformer architecture scales without
   code changes).
3. Replace demo placeholders with native-signer videos.
4. Spaced-repetition + accounts.
5. Real fairness slices (skin tone via Monk scale, handedness, lighting,
   camera angle).
6. WCAG 2.2 AA audit.
7. **CI is already in place** — same machinery runs against real data.

The architecture is now proven end-to-end for **both** static and dynamic
recognition, with **CI enforcing all release gates**. Each remaining v1 step
is a swap, not a rewrite.
