ML pipeline · static

MLP classifier (sklearn → ONNX)

24-class MLP. Eval is collision-aware: synth letters that share a finger pattern (A=E=M=N=S=T, etc.) are credited within their equivalence class, so the metric reports model skill not synth ambiguity. Strict 0.45 / collision-aware 0.93. Real data closes the gap.

5 · CI / Release gates
.github/workflows/ml-gate.yml

ML release gates in CI ⭐

One workflow runs both training scripts, the robustness sweep, and the JS↔Python parity test on every push. ci/check_nfr.py parses the markdown reports and exits non-zero on NFR-9 / NFR-10 / parity violations. Verified locally: gate FAILS correctly on the pre-augmentation (BEFORE) model and PASSES on the augmented one.

Why this matters

Reports → enforcement

The eval/robustness markdown files were already written by the training scripts. The CI gate makes them actionable: a regression in any slice blocks the merge. Same machinery will work unchanged on real WLASL/MS-ASL data.
