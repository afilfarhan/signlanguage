used in SignTutor v2 training. **CI will fail if a dataset path is used

## CI Gates

The following CI checks verify license compliance:

1. **License grep gate**: Every dataset path used in training must have an entry in this file.
   ```bash
   grep -q "$(basename $dataset_path)" ml/data/LICENSES.md || exit 1
   ```

2. **Signer-disjoint check**: `ci/check_signer_disjoint.py` verifies no signer appears in both train and test splits.

3. **Consent verification**: In-house recordings require signed consent forms before inclusion.

---

## Adding a New Dataset

When adding a new dataset, you must:

1. Add an entry to this file with all fields filled
2. Create a loader in `ml/data/{dataset_name}.py`
3. Add signer-disjoint split logic
4. Update `ci/check_nfr.py` to include the new dataset
5. Document the license type and any restrictions
6. Create a PR that includes this file update
